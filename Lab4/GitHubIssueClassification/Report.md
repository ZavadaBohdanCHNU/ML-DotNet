# Звіт до Лабораторної роботи №4

**Тема:** Класифікація проблем підтримки (GitHub Issues) за допомогою багатокласової класифікації з використанням ML.NET.

## 1. Мета роботи
Ознайомитися з алгоритмами багатокласової класифікації в ML.NET. Створити консольний застосунок на C#, який зможе автоматично розподіляти (прогнозувати) мітку області (`Area`) для нової проблеми на GitHub на основі її заголовку (`Title`) та опису (`Description`).

## 2. Теоретичні відомості
Класифікація — це завдання машинного навчання, яке відносить об'єкт даних до певного класу. Якщо кількість можливих класів більша за два, це називається **багатокласовою класифікацією** (Multiclass Classification). 
У даній роботі використано алгоритм **SdcaMaximumEntropy** — метод стохастичного подвійного координатного підйому, який добре підходить для класифікації текстових даних після їх векторизації.

## 3. Опис виконаних кроків
1. **Налаштування середовища та створення проекту:**
   - Створено консольний застосунок `GitHubIssueClassification`.
   - За допомогою `nuget.config` налаштовано локальне збереження NuGet-пакетів (папка `packages`), щоб проект не залежав від глобального кешу системи та легко переносився.
   - Встановлено базовий пакет `Microsoft.ML`.

2. **Підготовка даних (Data Preparation):**
   - Створено директорії `Data` та `Models`. Завантажено тренувальний (`issues_train.tsv`) та тестовий (`issues_test.tsv`) набори даних.
   - Створено клас `GitHubIssue` для вхідних даних (описує структуру колонок `.tsv` файлу за допомогою атрибутів `[LoadColumn]`).
   - Створено клас `IssuePrediction` для результатів прогнозу з атрибутом `[ColumnName("PredictedLabel")]`.

3. **Побудова конвеєра даних (Data Pipeline):**
   - Для обробки текстових даних застосовано перетворення `FeaturizeText`, яке перетворило текст `Title` і `Description` у числові вектори.
   - Ознаки було об'єднано в один стовпець `Features` за допомогою `Concatenate`.
   - Прогнозована колонка `Area` була трансформована в числовий ключ `Label` (`MapValueToKey`).

4. **Навчання моделі (Training):**
   - До конвеєра даних додано алгоритм класифікації `SdcaMaximumEntropy`.
   - Дані пропущені через пайплайн за допомогою функції `Fit()`, яка виконала навчання моделі на тренувальному датасеті.
   - Проведено одиночне прогнозування (`Predict`) для перевірки щойно натренованої моделі (результат: `area-System.Net`).

5. **Оцінка моделі (Evaluation):**
   - Використовуючи тестовий набір даних (`issues_test.tsv`), було проведено тестування моделі для визначення її ефективності.
   - Отримано наступні ключові метрики:
     - **MicroAccuracy:** 0.737 (Мікро-точність; чим ближче до 1, тим краще)
     - **MacroAccuracy:** 0.668 (Макро-точність; усі класи рівнозначні)
     - **LogLoss:** 0.915 (Логарифмічні втрати; чим ближче до 0, тим краще)
     - **LogLossReduction:** 0.645

6. **Збереження та використання (Deployment):**
   - Готову модель було серіалізовано та збережено у файл `model.zip` (функція `mlContext.Model.Save`).
   - Для демонстрації роботи модель було завантажено з файлу (`mlContext.Model.Load`) та створено новий `PredictionEngine`.
   - Для тестового запиту про помилку *Entity Framework* система успішно передбачила категорію: `area-System.Data`.

## 4. Висновок
В результаті виконання лабораторної роботи розроблено повноцінний ML.NET застосунок для класифікації текстових даних. Модель успішно навчається на основі текстових ознак за допомогою векторизації (`FeaturizeText`) та алгоритму багатокласової класифікації `SdcaMaximumEntropy`. Точність моделі (Micro Accuracy) становить ~73.7%, що є хорошим показником для цієї задачі.
