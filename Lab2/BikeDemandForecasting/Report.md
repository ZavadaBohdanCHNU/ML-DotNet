# Звіт з лабораторної роботи №2
**Тема:** Прогнозування попиту на сервіс оренди велосипедів за допомогою аналізу часових рядів та ML.NET

## 1. Мета роботи
Ознайомитися з основами прогнозування часових рядів за допомогою фреймворку ML.NET. Навчитися завантажувати дані з SQL-бази даних, створювати модель прогнозування за допомогою алгоритму Singular Spectrum Analysis (SSA), оцінювати її якість та використовувати для реальних прогнозів.

## 2. Архітектура рішення та особливості реалізації

### 2.1. Контейнеризація бази даних (Docker)
Замість локального екземпляра бази даних `LocalDB`, який використовується за замовчуванням у туторіалі Microsoft, було розгорнуто повноцінний **SQL Server 2022 у Docker-контейнері**.
- Налаштовано `docker-compose.yml` для розгортання образу `mcr.microsoft.com/mssql/server:2022-latest`.
- Створено PowerShell-скрипт `setup_db.ps1`, який автоматично завантажує файл БД `DailyDemand.mdf`, копіює його в контейнер та підключає до SQL Server (використовуючи `sqlcmd` та команду `CREATE DATABASE ... FOR ATTACH_REBUILD_LOG`).

### 2.2. Налаштування проекту та локальні NuGet-пакети
Проект створено як `.NET Console Application`. Для оптимізації дискового простору та уникнення засмічення глобального кешу комп'ютера, було створено файл `nuget.config`, який вказує завантажувати всі пакети (у тому числі важкі залежності ML.NET) у локальну папку `packages` безпосередньо всередині проекту.

Використані ключові пакети:
- `Microsoft.ML` (основний фреймворк машинного навчання)
- `Microsoft.ML.TimeSeries` (алгоритми для аналізу часових рядів)
- `System.Data.SqlClient` (драйвер для роботи з SQL Server)

## 3. Опис основних етапів роботи коду

### Крок 1. Завантаження даних
Дані завантажуються безпосередньо з SQL Server за допомогою `CreateDatabaseLoader`. Було сформовано SQL-запит, який зчитує поля `RentalDate`, `Year` та `TotalRentals`. Для колонок `Year` та `TotalRentals` застосовано приведення типів (`CAST(… as REAL)`), оскільки алгоритми ML.NET вимагають використання типу `Single` (чисел із плаваючою крапкою одинарної точності). 
Далі датасет було розділено на навчальну (`firstYearData`) та тестову (`secondYearData`) вибірки за значенням колонки `Year`.

### Крок 2. Побудова пайплайну та тренування моделі
Для прогнозування використано метод **Singular Spectrum Analysis (SSA)**. Це алгоритм, який розкладає часовий ряд на спектральні компоненти (тренд, шум, сезонність).

Параметри пайплайну `ForecastBySsa`:
- `windowSize: 7` — щотижневе вікно аналізу (для прогнозування використовуються дані 7 попередніх днів).
- `seriesLength: 30` — періодичність даних (місячний цикл).
- `trainSize: 365` — розмір навчальної вибірки (365 днів першого року).
- `horizon: 7` — прогнозування на 7 періодів (днів) вперед.
- `confidenceLevel: 0.95f` — 95% рівень довіри для обчислення верхньої та нижньої межі (LowerBound / UpperBound).

Модель натреновано викликом `forecastingPipeline.Fit(firstYearData)`.

### Крок 3. Оцінка (Evaluation) моделі
У методі `Evaluate()` перевірено якість роботи моделі на даних другого року. Зіставляючи реальні значення з прогнозами, вирахувано дві метрики якості:
- **Mean Absolute Error (MAE)**: показує абсолютне середнє відхилення прогнозу від факту (чим ближче до 0, тим краще).
- **Root Mean Squared Error (RMSE)**: середньоквадратична помилка (більше "штрафує" модель за великі відхилення).

### Крок 4. Збереження моделі
Для того, щоб не тренувати модель щоразу, її збережено у файл `MLModel.zip` за допомогою методу `CheckPoint()` об'єкта `TimeSeriesPredictionEngine`.

### Крок 5. Прогнозування (Forecasting)
Метод `Forecast()` здійснює прогнозування для наступних 7 днів. Для кожного дня виводиться безпосередньо прогнозоване значення оренди велосипедів (`Forecast`), а також верхня (`Upper Estimate`) та нижня (`Lower Estimate`) оцінки.

Після запуску додатку в консоль було виведено такі метрики, які повністю збігаються з еталонними результатами лабораторної роботи:

**Висновок:** Побудована модель не прогнозує абсолютно точну кількість оренд до одиниці, однак вона формує достовірний "коридор" (між нижньою та верхньою межею), який дозволяє сервісу оренди ефективно управляти ресурсами та планувати кількість необхідних велосипедів. Архітектурно застосування Docker.
