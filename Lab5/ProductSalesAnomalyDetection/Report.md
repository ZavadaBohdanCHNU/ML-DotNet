# Звіт до Лабораторної роботи: Виявлення аномалій у продажах (ML.NET)

## 1. Мета роботи
Навчитися будувати консольний додаток на базі ML.NET для аналізу часових рядів та виявлення двох типів аномалій у даних про продажі продуктів:
1. **Сплесків (Spikes)** — раптових, але тимчасових відхилень.
2. **Точок змін (Change points)** — початку постійних, тривалих змін у системі або тренді.

## 2. Інструментарій та вхідні дані
- **Мова програмування:** C# (.NET)
- **Бібліотеки:** `Microsoft.ML`, `Microsoft.ML.TimeSeries`
- **Набір даних:** `product-sales.csv` (на основі датасету "Shampoo Sales Over a Three Year Period"). Набір складається з 36 записів і містить два стовпці: `Month` та `ProductSales`.

## 3. Опис процесу реалізації

### Крок 1. Моделі даних
Було створено два класи:
- `ProductSalesData` — відповідає за завантаження вхідних даних (місяць та обсяг продажів).
- `ProductSalesPrediction` — містить результати прогнозування. Це масив `double[]` розміром 3 (для сплесків) або 4 (для точок змін), який містить: `Alert` (сигнал тривоги, 1 = аномалія, 0 = норма), `Score` (значення), `P-Value` (ймовірність), і `Martingale value` (лише для точок змін).

### Крок 2. Налаштування контексту (MLContext)
Ініціалізовано `MLContext`, який є стартовою точкою для всіх операцій ML.NET. Дані завантажені за допомогою `LoadFromTextFile`. На відміну від класичного машинного навчання, для виявлення аномалій в часових рядах не потрібен окремий етап "навчання" на великому масиві історичних даних з мітками — трансформери працюють безпосередньо з вхідними даними.

### Крок 3. Виявлення сплесків (Spike Detection)
Для виявлення тимчасових сплесків використано алгоритм `DetectIidSpike`.
- **Confidence (Довірчий інтервал):** 95. Визначає чутливість моделі.
- **PvalueHistoryLength:** Розмір ковзного вікна для обчислення ймовірності (в нашому випадку `docSize / 4 = 36 / 4 = 9`). Чим менше значення, тим швидше модель "забуває" попередні сплески.
**Пояснення для викладача:** P-Value показує ймовірність того, що точка є нормальною. Що ближче значення до 0, то вища ймовірність аномалії.

### Крок 4. Виявлення точок змін (Change Point Detection)
Для виявлення постійних змін використано алгоритм `DetectIidChangePoint`. 
На відміну від сплесків, зміна фіксується з невеликою затримкою, оскільки алгоритм повинен переконатися, що відхилення є постійною зміною, а не просто випадковим сплеском.
- **Martingale value:** Додаткова метрика, яка дозволяє визначити, наскільки "дивною" є точка даних на основі послідовності минулих P-значень.

## 4. Результати роботи програми

Після запуску `dotnet run` програма проаналізувала 36 записів:

**Виявлені сплески (Spikes):**
- Продажі: `341.50` (P-Value: 0.00) <-- Spike detected
- Продажі: `426.60` (P-Value: 0.00) <-- Spike detected
- Продажі: `580.50` (P-Value: 0.00) <-- Spike detected
- Продажі: `687.00` (P-Value: 0.01) <-- Spike detected

**Виявлені точки змін (Change points):**
- Продажі: `197.80` (P-Value: 0.48, Martingale value: 44.23) <-- alert is on, predicted changepoint

## 5. Висновок
Під час виконання лабораторної роботи було успішно застосовано механізми аналізу часових рядів ML.NET. 
Модель успішно виявила 4 різкі сплески продажів (разові аномалії) та одну точку зміни (початок нового довгострокового тренду чи рівня продажів), використовуючи статистичні алгоритми IID.
